# GitHub Repo Exposure — Full Checklist

Detailed actionable checklist for each phase. Copy and use per-repo.

## Phase 1: GitHub On-Site SEO

```
On-Site SEO:
- [ ] repo name contains 2-3 core keywords
- [ ] about/description starts with action verb, includes keywords, under 350 chars
- [ ] topics filled (target 20 tags)
  - [ ] language tags (python, cpp, rust, etc.)
  - [ ] domain tags (llm, compiler, web, etc.)
  - [ ] tool/ecosystem tags (pytorch, gh-actions, docker, etc.)
  - [ ] competitor/alternative tags
- [ ] README H1 contains project name with keywords
- [ ] badges added (build status, license, version)
- [ ] first 100 words contain core value proposition
- [ ] clear H2 structure (Features, Quick Start, Installation, Usage, etc.)
- [ ] quick start section with copy-paste example
- [ ] license file present
```

## Phase 2: Visual Branding

```
Visual Branding:
- [ ] social preview image uploaded (1280x640)
  - [ ] contains logo + project name + slogan
  - [ ] not the default GitHub preview
- [ ] demo GIF/animation at top of README
  - [ ] under 10 seconds
  - [ ] shows core functionality
  - [ ] loops cleanly
```

## Phase 3: Off-Site Backlinks

```
Backlinks:
- [ ] identified 3-5 relevant Awesome lists
  - [ ] awesome list 1: ___________ (PR: pending / merged)
  - [ ] awesome list 2: ___________ (PR: pending / merged)
  - [ ] awesome list 3: ___________ (PR: pending / merged)
- [ ] Reddit post written and submitted
  - [ ] subreddit: ___________
  - [ ] narrative style (not link drop)
- [ ] Hacker News submission
  - [ ] submitted at US Eastern 8-9 AM
- [ ] technical blog post published
  - [ ] platform: ___________
  - [ ] links back to repo
- [ ] documentation site deployed
  - [ ] platform: GitHub Pages / Read the Docs / other
  - [ ] URL: ___________
```

## Phase 4: Growth Hacking

```
Growth Hacking:
- [ ] commit cadence: at least weekly activity
- [ ] release created for latest features (semver)
- [ ] release notes written clearly
- [ ] README optimized for LLM discoverability
  - [ ] clear English what/who description
  - [ ] comparison with alternatives section
  - [ ] structured feature list / benchmarks
- [ ] community engagement setup
  - [ ] GitHub Discussions enabled
  - [ ] CONTRIBUTING.md added
  - [ ] issue response time < 24 hours
```

## Scoring (Self-Assessment)

After completing the checklist, rate the repo:

| Score | Level | Meaning |
|-------|-------|---------|
| 0-5 | bare minimum | repo exists but invisible |
| 6-10 | discoverable | shows up in targeted searches |
| 11-15 | competitive | ranks well in niche |
| 16-20 | strong presence | organic inbound traffic |
| 21+ | growth engine | self-sustaining visibility loop |

Count each checked box as 1 point.
